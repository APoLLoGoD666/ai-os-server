## 3. Integration Points

### 3.1 Vault File Integration

| Subsystem | Storage path | Write method | Conflict risk |
|-----------|-------------|-------------|--------------|
| episodic-memory.js | `12 Memory/Episodes/ep-{id}.json` | New file per episode | None — new files |
| goal-tracker.js | `System/Goals/goal-{id}.json` | New file per goal | None — new files |
| planning-quality-registry.js | `System/PlanQuality/plan-quality-registry.json` | Merge into array | Low — filter by `synthetic===true` |
| obsidian-memory.js | `01 Executive/Lessons.md` | Append between markers | Low — marker-delimited |
| memory-indexer.js | `12 Memory/memory-index.json` | Deleted during cleanup | Low — auto-rebuilt |
| improvement-executor.js | `System/Improvements/proposals.json` | Not directly written | None — generated by engine |
| self-evaluator.js | `System/Cognition/Evaluations/*.json` | Not directly written | None — generated by engine |

### 3.2 Supabase Integration

| Table | Cleanup key | Risk |
|-------|------------|------|
| `apex_agent_runs` | `task_id LIKE 'synth-%'` | None — isolated by prefix |
| `transactions` | `description LIKE '[SYNTHETIC]%'` | None — isolated by prefix |
| `invoices` | `invoice_number LIKE 'SYNTH-%'` | None — isolated by prefix |
| `email_threads` | `thread_id LIKE 'synth-thread-%'` | None — isolated by prefix |

### 3.3 In-Process Cache Integration

`episodic-memory.js` maintains an in-process LRU cache of 50 episodes. After vault file writes, the cache will not include new synthetic episodes until `loadFromDisk()` is triggered (server restart) or `storeEpisode()` is called via the API.

**Implication:** For validation checkpoints, trigger a server restart after loading synthetic data OR call `POST /api/memory/rebuild` if that endpoint exists.

## 4. Cleanup Strategy

### 4.1 Tier-specific vs. global cleanup

Each record includes `dataset_id` and `synthetic: true`. Cleanup supports:
- `cleanupSyntheticDataset('sdv1-dim')` — remove only Tier 1 records
- `cleanupSyntheticDataset('sdv1-loop')` — remove only Tier 2 records
- `cleanupSyntheticDataset('sdv1-scale')` — remove only Tier 3 records
- `cleanupSyntheticDataset('all')` — remove all synthetic data

### 4.2 Cleanup order (dependency-safe)

```
1. Delete ep-synth-*.json files (episodic memory)
2. Delete goal-synth-*.json files (goal tracker)
3. Filter plan-quality-registry.json (remove synthetic===true entries)
4. Strip synthetic markers from Lessons.md (regex delete between markers)
5. Delete memory-index.json (auto-rebuilt from remaining real episodes)
6. Delete synth-*.md in 13 Briefings/Conversations/ (chat history)
7. Delete synth-*.md in 02 Projects/Archive/ and Active/ (project history)
8. DELETE FROM apex_agent_runs WHERE task_id LIKE 'synth-%'
9. DELETE FROM transactions WHERE description LIKE '[SYNTHETIC]%'
10. DELETE FROM invoices WHERE invoice_number LIKE 'SYNTH-%'
11. DELETE FROM email_threads WHERE thread_id LIKE 'synth-thread-%'
```

### 4.3 Post-cleanup verification

```
episodeCount()          → 0
goalStats().total       → 0 (or pre-test count)
planQuality records     → 0
Lessons.md              → "None yet" (or pre-test content only)
adaptation-registry     → unchanged (no synthetic records written directly)
apex_agent_runs COUNT   → 0 synthetic rows
transactions COUNT      → 0 synthetic rows
```

## 6. Risk Analysis

| Risk | Severity | Likelihood | Mitigation |
|------|----------|-----------|------------|
| Synthetic lessons survive cleanup if markers corrupted | Medium | Low | Lessons also prefixed `[SYNTHETIC:{dataset_id}]` for text-search fallback |
| plan-quality-registry.json overwritten by real `assignWork()` during test | Low | Low | Cleanup filters by `synthetic===true`, leaves real records intact |
| episodic-memory in-process cache serves stale data after cleanup | Low | Medium | Server restart required after cleanup (documented in rollback procedure) |
| Supabase service-role key wrong/expired | Medium | Very low | Validator runs SELECT checks; immediate feedback on failure |
| Memory index not rebuilt after load | Medium | Medium | Validator checks `rebuildIndex()` trigger; CLI prompts server restart |
| Goal files interfering with real goal completion metrics | Medium | Low | Cleanup removes all `goal-synth-*.json`; goal-tracker reads entire directory |
| Tier 2 load without Tier 1 (out-of-order) | High | Low | CLI `loadTier2` calls `loadTier1` first (idempotent file writes) |
| Duplicate records on re-load | Low | Medium | Episode/goal files are deterministic names — overwrite is safe |
| Lessons.md double-marker on re-load | Medium | Medium | Loader checks for existing marker before appending; skips if present |

node test-data-generator/cli.js load tier3

# Validate loaded data
node test-data-generator/cli.js validate tier1
node test-data-generator/cli.js validate tier2
node test-data-generator/cli.js validate tier3

# Show current synthetic data counts
